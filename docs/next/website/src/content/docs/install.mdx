---
title: Install Herdr
description: Build, install, and verify Herdr on Linux and macOS.
---

The kazuph/herdr fork is installed from source. A manually selected binary from [fork releases](https://github.com/kazuph/herdr/releases) is also supported. Do not use upstream release binaries, `herdr.dev` installers, Homebrew, mise, or automatic release downloads for this fork runtime.

## Install

Build the fork and install the resulting binary:

```bash
git clone https://github.com/kazuph/herdr
cd herdr
just build
install -m 755 target/release/herdr ~/.local/bin/herdr
```

On macOS local development installs, sign the binary after copying it:

```bash
codesign -s - -f ~/.local/bin/herdr
```

For alternate-binary testing, copy the same build under another name such as `herdrx`; that binary uses its own config, state, socket, and session namespace.

## Verify

Start Herdr:

```bash
herdr
```

If your shell cannot find `herdr`, restart the terminal or check that the install directory is on your PATH.

## Update

Self-update is disabled in this fork. Update by rebuilding from the fork checkout and replacing the local binary explicitly:

```bash
just build
install -m 755 target/release/herdr ~/.local/bin/herdr
```

Restarting a running Herdr server or client is a separate operation. Do not stop or replace the active runtime implicitly just because a new binary was built.

## Requirements

Build requirements are the Rust toolchain and the platform libraries required by this source tree.
